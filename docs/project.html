Python Quiz Engine: Context-Aware AI Learning Assistant

Static practice assessments lack targeted feedback mechanisms when a user answers incorrectly. This production service closes that learning loop by serving dynamically generated Python data structure and algorithm challenges alongside a real-time LLM feedback pipeline. Utilizing low-latency API inference, the engine instantly evaluates user responses and injects contextual code breakdowns to explain fundamental computer science errors.

QIntel: Intelligent Real-Time Audience Processing System

Executive Abstract

Large meetings often suffer from repeated questions and missed answers. This system reduces redundancy, improves clarity, and enhances the experience for both speakers and participants. AI module detects duplicate questions using text embeddings, classifies questions by topic, suggests answers based on historical responses, and prevents repetitive submissions.QIntel solves chaos in the Q&A events. The engine help presenters with a clean dashboard to filter and display high-value questions on screen.

🚀 Launch Application (Full Screen) 📄 Technical Spec (.pdf) 📊 Architecture Deck (.pdf)

Engineering Specifications

  • Active AI Module Integration: Building a transformer-based text embedding pipeline running cosine similarity matrix thresholds. This identifies incoming duplicate questions in the buffer cache, prevents database pollution, and auto-groups queries by semantic context.
  • Core Architecture Stack: Built via an R/Shiny reactive model, leveraging an integrated SQLite database layer for persistence and optimized text matching libraries.

Flight Delay Diagnostics: Propagated Weather Risk Modeling (DFW)

Executive Abstract

A predictive modeling project analyzing weather patterns, operational factors, and propagated delays to identify airport pairs that should not share a pilot due to weather-related delays.I used Lasso, Random Forest for methodologies for this project.

💻 Explore Engine Source Code on GitHub

Engine Architecture & Methodology

To analyze complex microclimate interactions and operational constraints at Dallas-Fort Worth International Airport, the data modeling pipeline executes two primary algorithmic frameworks: 1. Lasso Regression (L1 Regularization): Applied to perform automated feature pruning over highly dimensional weather arrays, shrinking non-impact parameters to absolute zero. 2. Random Forest Regression Engines: Implemented to capture non-linear cascading delay variables and localized airport pair bottleneck thresholds.

Technologies Utilized: R, tidyverse, glmnet (Lasso), randomForest.

Relational Player Metrics Index & Content Delivery Engine

Executive Abstract

This application provides a dynamic interface to explore NBA player data stored in a relational database. By integrating a SQLite backend with a Shiny front-end, I created a tool that allows for real-time data filtering and image rendering.

🚀 Launch Application (Full Screen) 💻 View Repository on GitHub

System Mechanics

  • Data Layer: Relational indexes built natively in SQLite to optimize lookups over multi-variable player frames.
  • Presentation Layer: Integrated with DT (DataTables) rendering pipes to execute clean server-side table pagination and multi-parameter filtering queries.

Technologies Utilized: R, Shiny, and SQLite.
